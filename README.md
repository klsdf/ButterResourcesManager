# 需求背景

随着互联网发展，硬件的存储价格不断降低，使得个人拥有大容量存储成为可能。因此也滋生了越来越多的“仓鼠症”。很多仓鼠症患者在收集的资源越来越多的同时，也会发现管理/整合资源的难度越来越高。在多媒体的领域，这个问题尤为严重。
笔者对这个问题也是头疼已久，当资源超过10TB之后，无论是按照种类区分，还是按照常用程度区分，都无法很好地满足笔者的需求。
因此开发一个资源管理器是很有必要的。

## 目标用户定位

1. 拥有强烈仓鼠症的用户
2. 希望以更优雅方式管理软件和媒体的用户

## ✨ 项目特色

### 🎯 一站式多媒体管理

- **统一界面**：一个应用管理所有类型的多媒体资源
- **无缝切换**：游戏、漫画、视频、音频、小说、网站统一管理
- **智能分类**：自动识别和分类不同类型的媒体文件

### 🔍 强大的筛选系统

- **多维度筛选**：支持标签、作者、系列、分类等多重筛选
- **实时搜索**：快速定位目标资源
- **智能排序**：多种排序方式满足不同需求

### 🎨 优雅的用户体验

- **现代化UI**：基于Vue 3的响应式界面设计
- **主题切换**：支持亮色/暗色主题
- **组件化设计**：统一的UI组件和交互体验

### 📱 内置播放器/阅读器

- **游戏启动器**：直接启动和管理游戏
- **漫画阅读器**：支持缩放、全屏、键盘控制
- **视频播放器**：内置播放器，记录观看进度
- **音频播放器**：支持多种音频格式
- **小说阅读器**：可调节字体和阅读设置

### 💾 智能数据管理

- **本地存储**：所有数据本地存储，保护隐私
- **数据备份**：支持数据导入导出和备份
- **统计功能**：详细的播放/阅读统计
- **进度记录**：自动记录各种进度信息

## 技术栈

- **前端框架**: Vue 3
- **构建工具**: Vite
- **桌面框架**: Electron
- **包管理器**: npm
- **开发语言**: JavaScript/TypeScript

## 快速开始

### 安装依赖

```bash
npm install
```

### 开发模式

启动Vue开发服务器

```bash
npm run dev
```

启动Vue开发服务器和Electron：

```bash
npm run electron-dev
```

这将同时启动Vite开发服务器和Electron应用。

### 构建应用

构建Vue应用：

```bash
npm run build
```

构建Electron应用：

```bash
npm run electron-build
```

### 打包分发

创建可分发的应用包：

```bash
npm run dist
```
